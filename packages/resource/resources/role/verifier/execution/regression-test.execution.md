<execution id="regression-test">

<constraint>

## 回归检测约束

- **基线冻结**：每次角色迭代前必须冻结当前用例库为基线
- **差异报告**：迭代后对比基线，输出差异报告
- **不许静默回归**：任何 P0/P1 级别回归必须立即反馈
- **基线版本化**：基线必须按版本号持久化，便于回溯

</constraint>

<rule>

## 回归测试规则

1. **基线对比**：用迭代前的测试用例库逐项对比迭代后的输出
2. **变更识别**：识别角色定义文件、Prompt、工具的任何变更
3. **回归分级**：变更可能导致的回归分 P0-P3 等级
4. **基线更新**：通过验证后，更新基线到新版本

</rule>

<guideline>

## 回归测试场景分类

### 变更类型 vs 回归风险

| 变更类型 | 回归风险 | 测试范围 |
|---------|---------|---------|
| **Prompt 修改** | 中 | 全量回归 |
| **工具新增/替换** | 高 | 工具相关用例 + 集成用例 |
| **五维模型调整** | 高 | 五维验证用例 |
| **硬性规则变更** | 高 | 硬性规则相关用例 |
| **协作边界调整** | 中 | 协作路由用例 |
| **格式调整** | 低 | 冒烟测试 |
| **文档/注释调整** | 无 | 不需回归 |

### 回归用例优先级

| 优先级 | 用例类型 |
|--------|---------|
| P0 | 核心功能 + 硬性规则相关 |
| P1 | 主要场景 + 协作路由 |
| P2 | 边界场景 + 异常处理 |
| P3 | 风格细节 + 表达偏好 |

</guideline>

<process>

## 回归检测执行流程

### 流程 1：变更识别

```
1. 对比角色定义版本
   - 上一版本：git tag 或 PERSENG_HOME/role/<id>/.version
   - 当前版本：当前文件状态

2. 识别变更点
   - 文件变更（git diff）
   - Prompt 变更
   - 工具变更
   - 配置变更

3. 评估变更影响
   - 受影响的测试用例范围
   - 风险等级预估
```

### 流程 2：基线对比执行

```
1. 加载基线用例库
   - 路径：~/.perseng/verifier/baselines/<role-id>/<version>/
   - 包含：所有历史用例 + 上次运行结果

2. 对每个基线用例执行当前角色
   - 输入相同 prompt
   - 记录当前输出
   - 对比基线输出

3. 识别差异
   - 完全一致：✅ 无回归
   - 关键差异（影响功能）：❌ 回归
   - 风格差异（不影响功能）：⚠️ 风格漂移
   - 改进（新行为更优）：✅ 改进
```

### 流程 3：回归分级

```
对每个识别出的回归：

1. 评估影响范围
   - 影响核心功能 → P0
   - 影响主要场景 → P1
   - 影响边缘场景 → P2
   - 仅风格差异 → P3

2. 评估修复优先级
   - P0：阻塞发布，必须立即修复
   - P1：24 小时内修复
   - P2：一周内修复
   - P3：下个迭代修复

3. 给出修复建议
   - 是否回滚到上一版本？
   - 是否针对性修复当前版本？
   - 是否更新基线接受新行为？
```

### 流程 4：基线更新

```
1. 通过验证后
   - 归档旧基线（保留 3 个历史版本）
   - 创建新基线
   - 更新基线版本号

2. 输出更新报告
   - 新基线版本
   - 用例数变化
   - 历史对比
```

</process>

<criteria>

## 回归测试评价标准

### 通过标准

- ✅ 0 个 P0 回归
- ✅ 0 个 P1 回归
- ✅ P2 回归有明确的修复计划
- ✅ P3 回归记录到下个迭代

### 回归 vs 改进的判定

| 现象 | 判定 | 处理 |
|------|------|------|
| 新输出与基线完全一致 | 无回归 | 更新基线 |
| 新输出优于基线（如更准确） | 改进 | 更新基线 |
| 新输出与基线功能等价但表述不同 | 风格漂移 | 评估是否接受 |
| 新输出缺失基线的关键能力 | 回归 | 必须修复 |
| 新输出违反基线的硬性规则 | 严重回归 | 阻塞，立即修复 |

### 回归测试的频率

| 触发条件 | 测试范围 |
|---------|---------|
| **每次角色 Prompt 修改** | 全量回归（≥ 30 用例） |
| **每次工具更新** | 工具相关回归（≥ 10 用例） |
| **每次发布** | 全量回归 + 冒烟测试 |
| **每周巡检** | 抽样回归（≥ 10 用例） |
