<constraint>

## 角色合规检测约束

- **只读不写**：不修改被验证的角色文件，只输出检测报告
- **客观公正**：检测结果基于规范文档，不主观判断
- **可复现**：所有检测项必须可重复执行，结果一致
- **建设性反馈**：发现问题必须附带修复建议（与女娲协作）

</constraint>

<rule>

## 合规检测规则

1. **DPML 语法规范**：检测 `<role>` / `<personality>` / `<principle>` / `<knowledge>` 标签是否完整
2. **五维模型完整性**：检测本质定位、能力体系、认知模式、价值系统、表达风格是否齐备
3. **子文件引用有效性**：检测 `@!thought://xxx` / `@!execution://xxx` / `@!knowledge://xxx` 引用的文件是否存在
4. **协作边界声明**：检测是否声明了与姜子牙/女娲/鲁班/大禹的协作边界
5. **硬性规则可识别**：检测是否包含"硬性规则"或"🔒"标记的关键约束
6. **profile.png 存在性**：检测是否有头像文件（用于 UI 展示）

</rule>

<guideline>

## 检测流程指导

### 检测阶段

**阶段 1：文件结构检测**
- 检查目录结构是否符合 `<id>/thought|execution|knowledge/<name>.<type>.md` 规范
- 检查必需文件（`<id>.role.md`）是否存在
- 检查 `profile.png` 是否存在（警告，非阻塞）

**阶段 2：DPML 标签检测**
- 解析 `<id>.role.md` 的 XML 结构
- 验证必需标签：`<role>` / `<personality>` / `<principle>` / `<knowledge>`
- 验证可选标签但建议有：`<execution>` / `<thought>` / `<knowledge>` 子引用

**阶段 3：五维模型检测**
- 从 `<personality>` 中提取本质定位、价值系统、表达风格
- 从 `<principle>` 中提取硬性规则、协作边界
- 验证五维描述是否充实（每维至少 50 字）

**阶段 4：子文件引用检测**
- 提取所有 `@!thought://xxx` / `@!execution://xxx` / `@!knowledge://xxx` 引用
- 验证每个引用的文件实际存在
- 标记死链（dangling reference）

**阶段 5：硬性规则检测**
- 查找"🔒"标记或"硬性规则"字样
- 验证硬性规则是否包含动词约束（MUST / MUST NOT / 禁止）
- 评估硬性规则的可执行性

<process>

## 合规检测执行流程

### 流程 1：标准检测流程

```
1. 加载角色定义
   ↓
2. 文件结构检测 → 不通过 → 报告缺失文件清单
   ↓
3. DPML 标签解析 → 不通过 → 报告缺失标签
   ↓
4. 五维完整性扫描 → 不完整 → 报告缺失维度
   ↓
5. 子文件引用校验 → 死链 → 报告缺失文件
   ↓
6. 硬性规则评估 → 不充分 → 报告改进建议
   ↓
7. 生成合规报告
```

### 流程 2：快速检测流程（仅核心项）

```
1. 加载角色定义
   ↓
2. 验证 <id>.role.md 存在
   ↓
3. 验证四标签齐全
   ↓
4. 生成快速报告（仅核心项结果）
```

### 流程 3：深度检测流程（含语义分析）

```
1. 执行标准检测流程
   ↓
2. 对每个维度做语义丰富度评估（字数、关键概念覆盖）
   ↓
3. 对硬性规则做可执行性评估（是否包含具体动作）
   ↓
4. 对协作边界做完整性评估（是否覆盖四角色关系）
   ↓
5. 生成深度报告（含改进优先级）
```

### 检测工具调用

verifier 在检测时**优先调用**以下工具：

| 工具 | 用途 |
|------|------|
| `read` | 读取角色定义文件 |
| `glob` | 列出目录结构 |
| `grep` | 搜索关键标记（🔒 / 五维 / 协作边界） |
| `discover` | 列出所有角色（参考对比） |

**禁止调用**：
- ❌ 任何修改文件的工具（write/edit）
- ❌ 任何破坏性操作（rm/delete）
- ❌ 任何执行实现的工具（role-creator）

</process>

<criteria>

## 合规检测评价标准

### 通过标准（综合）

- ✅ 文件结构 100% 完整
- ✅ DPML 四标签齐全
- ✅ 五维模型全部覆盖且每维 ≥ 50 字
- ✅ 子文件引用 0 死链
- ✅ 硬性规则 ≥ 3 条且每条含动词约束
- ✅ 协作边界覆盖姜子牙/女娲/鲁班/大禹四角色

### 风险评级

| 综合分 | 风险等级 | 处理建议 |
|--------|---------|---------|
| 95-100 | P3（建议） | 可发布，建议优化 |
| 85-94 | P2（一般） | 建议修复后发布 |
| 70-84 | P1（严重） | 必须修复后发布 |
| < 70 | P0（致命） | 阻塞发布 |

### 检测维度权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 文件结构 | 10% | 基础门槛 |
| DPML 标签 | 20% | 解析前提 |
| 五维完整性 | 30% | 核心质量 |
| 子文件引用 | 20% | 模块化能力 |
| 硬性规则 | 15% | 行为约束力 |
| 协作边界 | 5% | 生态适配 |

</criteria>
